# Как запустить сайт на GitHub и получать заявки

Это статичный лендинг: сервер не нужен. GitHub Pages просто раздаёт ваши HTML/CSS/JS как обычные файлы.

---

## Шаг 1. Регистрация на GitHub (если ещё нет аккаунта)

1. Зайдите на https://github.com
2. Нажмите **Sign up** и создайте аккаунт

---

## Шаг 2. Создать репозиторий и залить сайт

### Вариант А: через сайт GitHub

1. Нажмите **+** в правом верхнем углу → **New repository**
2. **Repository name:** например `remont-saratov` (латиница, без пробелов)
3. Отметьте **Public**
4. Нажмите **Create repository**

5. Нажмите **uploading an existing file**
6. Перетащите в окно браузера **всё содержимое** папки `site`:
   - `index.html` (в корень!)
   - папку `css` с файлом `styles.css`
   - папку `js` с файлом `app.js`
   - файл `logo.svg`

7. Внизу страницы нажмите **Commit changes**

### Вариант Б: через Git (если установлен)

```bash
cd c:\Users\user\pupk\site
git init
git add .
git commit -m "Лендинг СаратовТехСервис"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/remont-saratov.git
git push -u origin main
```

(Замените `ВАШ_ЛОГИН` и `remont-saratov` на ваши данные.)

---

## Шаг 3. Включить GitHub Pages

1. В репозитории откройте **Settings** (Настройки)
2. В левом меню найдите **Pages**
3. В блоке **Source** выберите **Deploy from a branch**
4. В **Branch** выберите `main` и папку `/ (root)`
5. Нажмите **Save**

Через 1–2 минуты сайт будет доступен по адресу:

```
https://ВАШ_ЛОГИН.github.io/remont-saratov/
```

(Замените на ваш логин и имя репозитория.)

---

## Шаг 4. Настроить приём заявок (Formspree)

GitHub Pages — только статика, сервер не запускается. Чтобы получать заявки, используем **Formspree** — сервис, который принимает данные формы и отправляет их на вашу почту.

### 4.1. Регистрация

1. Зайдите на https://formspree.io
2. Нажмите **Get Started** и зарегистрируйтесь (можно через Google)

### 4.2. Создать форму

1. Нажмите **+ New Form**
2. Название: например «Заявки с сайта»
3. Email: ваша почта, куда будут приходить заявки
4. Нажмите **Create Form**

### 4.3. Скопировать адрес формы

После создания Formspree покажет адрес вида:

```
https://formspree.io/f/xxxxxxxx
```

Скопируйте его. Часть `xxxxxxxx` — это **ID формы**.

### 4.4. Подставить ID в проект

Откройте файл **`js/app.js`** в любом редакторе (Блокнот, VS Code и т.п.).

Найдите в начале файла строку:

```javascript
const FORMSPREE_ID = 'YOUR_FORM_ID';
```

Замените `YOUR_FORM_ID` на ваш ID из адреса Formspree (часть после `/f/`).

Например, если адрес `https://formspree.io/f/abc123xy`, то должно быть:

```javascript
const FORMSPREE_ID = 'abc123xy';
```

Сохраните файл и загрузите обновлённый `app.js` в репозиторий на GitHub. После этого заявки начнут приходить на указанный email.

---

## Шаг 5. Обновить ссылки на ваш реальный адрес (по желанию)

Когда сайт уже опубликован, откройте `index.html` и замените `example.github.io/remont-saratov` на ваш реальный адрес:

1. **Canonical** — найдите строку `rel="canonical"` и укажите ваш URL:
   ```html
   <link rel="canonical" href="https://ВАШ_ЛОГИН.github.io/remont-saratov/">
   ```

2. **Schema.org** — в конце файла в блоке `<script type="application/ld+json">` найдите `"url"` и пропишите тот же адрес.

Это нужно для SEO; на работу сайта не влияет.

## Краткая сводка

| Действие              | Где / что делать                          |
|-----------------------|-------------------------------------------|
| Хостинг               | GitHub Pages (бесплатно)                  |
| URL сайта             | `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО`   |
| Заявки на почту       | Formspree (бесплатно до 50 в месяц)      |
| Сервер / база данных  | Не нужны                                 |

Если что-то не получается — напишите, на каком шаге застряли.
